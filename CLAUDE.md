# Swappa — Claude Instructions

## Before starting any new task

**Always pause before writing code or making changes.** For every new feature request or task, do the following first:

### 1. Ask scoping questions (pick 2–4 that are most relevant)

- What is the intended user flow for this feature?
- Who is the target user, and what is their context when using this?
- Are there edge cases or constraints to consider (e.g. mobile, offline, accessibility, performance)?
- What does "done" look like — any acceptance criteria or success metrics?
- Should this integrate with any existing features or data?
- Are there design preferences (layout, color, tone, interaction style)?

### 2. Proactively suggest ideas (offer 1–3 unsolicited)

Before implementing, surface ideas such as:
- Adjacent features that would complement what's being built
- UX patterns that would improve the experience
- Technical approaches that reduce complexity or improve performance
- Potential pitfalls worth avoiding upfront

## Why this matters

Swappa is an early-stage product. Scope and direction are still being shaped collaboratively. Asking questions upfront avoids wasted rework, and surfacing ideas early helps build better features. The user wants idea-driven collaboration, not narrow task execution.

## How to apply

At the start of every **new task** (not mid-task): ask 2–4 targeted questions and offer 1–3 suggestions **before writing any code**. Keep it concise — a short exchange, not a long interview.
